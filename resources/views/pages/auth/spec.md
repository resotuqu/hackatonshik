Полная спецификация Hackatonshik
1. Общее описание
Hackatonshik — это веб-платформа, предназначенная для создания, управления и участия в хакатонах.
Платформа разделяет пользователей на две основные роли: Организатор и Участник.
2. Технологический стек
Backend: Laravel 13
База данных: PostgreSQL
Frontend: Blade, Tailwind CSS, DaisyUI
Аутентификация: Yandex OAuth, VK OAuth, собственная регистрация
SMS-подтверждение телефона: Plusofon API
Дополнительные пакеты:
spatie/media-library (работа с изображениями и документами)
spatie/laravel-markdown (поддержка Markdown)
laravel/socialite (OAuth)
3. Поля регистрации (обязательные)
ФИО (фамилия, имя, отчество)
Дата рождения
Номер телефона (с подтверждением через SMS)
Адрес электронной почты
Никнейм (уникальный)
Публичный доступ
Публичный список всех опубликованных хакатонов (главная страница / каталог) с фильтрами и поиском.
Публичные профили участников (доступны по уникальному никнейму всем пользователям, даже неавторизованным).
4. Функционал Организатора
Создание нового хакатона, включая:

Наименование
Описание в формате Markdown
Несколько прикреплённых изображений (обложка + галерея)
Информационные документы (загрузка)
Документы для обязательного заполнения участниками (шаблоны для загрузки)
Требования к ссылкам по кейсам: организатор создаёт список обязательных ссылок для каждого кейса (например: «Фигма», «Репозиторий GitHub», «Демонстрационное видео» и т.д.) с наименованием и опциональным описанием
Даты и таймлайн: дата начала/окончания регистрации, самого хакатона, дедлайны подачи решений, питч-сессии и кастомные дедлайны
Возможность создания этапов хакатона (регистрация → отбор → работа → судейство → награждение). Каждый этап имеет название, описание (Markdown), даты начала/окончания, порядок
Статусы хакатона: draft → published → registration_open → in_progress → judging → finished → archived (автоматическое и ручное управление)
Формат хакатона: онлайн (ссылки на ресурсы: Zoom, Discord, Google Meet и т.д.) или оффлайн (адрес, город, дополнительное описание: проезд, парковка, требования)
Возрастные ограничения (минимальный / максимальный возраст — автоматическая проверка по дате рождения)
Количество участников (опционально)
Количество участников в одной команде (от … до …)
Предлагаемые роли в команде и их категории (разработчик, дизайнер и т.д.)
Кейсы для участия
Ссылки на социальные сети хакатона
Итоги хакатона (призовые места по кейсам)

Создание объявлений для хакатона (Markdown-текст, отображается на странице хакатона, страницах команд и в уведомлениях участников).
Просмотр списка всех своих хакатонов.
Просмотр статистики хакатона (количество команд/участников, конверсия по этапам, заполненность документов).
Просмотр среднего рейтинга хакатона (на основе отзывов участников).
Приём и отклонение заявок команд.
Просмотр и скачивание подписанных документов участников.
Экспорт: участников и команд в Excel; всех подписанных документов в ZIP-архив.
Загрузка сертификатов для каждого участника (с генерацией PDF и автоподписью).
5. Функционал Участника
Создание команды:

Название команды
Обложка команды
Описание команды (Markdown)
Роли в команде, навыки ролей, категория роли
Ссылки на социальные сети команды

Подача заявки команды на конкретный хакатон (явный шаг после создания команды).
Подача заявки на роль в существующую команду.
Загрузка заполненных обязательных документов.
Загрузка обязательных ссылок по кейсам (любой участник команды может загрузить ссылки, запрошенные организатором: Фигма, GitHub и т.д.).
Для капитана команды: приём и отклонение заявок на роли.
Просмотр своих команд (с поиском и фильтрами вакансий внутри хакатона по ролям, навыкам и категориям).
Просмотр истории участия в хакатонах.
Публичный профиль участника (по никнейму): история участия, список сертификатов, статистика (количество участий, призовые места).
После завершения хакатона: возможность оставить отзыв организатору (рейтинг 1–5 + комментарий в Markdown).
Просмотр и скачивание своих сертификатов (PDF с автоподписью).
Редактирование профиля: Markdown-описание, подтверждение телефона, изменение настроек.

7. Важные требования
Все текстовые описания хранятся и отображаются в формате Markdown.
Поддержка множественной загрузки изображений и документов (spatie/media-library).
Строгая проверка прав доступа (Policies).
Статусы заявок: pending, approved, rejected.
Уведомления: веб-интерфейс (иконка в шапке с раскрывающимся списком), Email (Laravel Mail). Организаторские объявления попадают в уведомления и на страницы хакатона/команд.
Генерация PDF-сертификатов с возможностью автоподписи.
Экспорт участников/команд в Excel и документов в ZIP.
Автоматическая проверка возрастных ограничений при подаче заявки.
Поиск и фильтры вакансий в командах внутри хакатона.
Автоматический расчёт среднего рейтинга хакатона.
Публичные профили участников и отзывы (только после завершения хакатона, один раз).
Адаптивность и отзывчивость интерфейса: полная поддержка всех устройств — ПК, планшеты, телефоны. Мобильная-first вёрстка на базе Tailwind CSS + DaisyUI с responsive-классами. Обязательное тестирование на реальных устройствах.
Автоматическое изменение статусов хакатона по датам (cron).
Валидация: нельзя опубликовать хакатон без дат и этапов.
8. Цель проекта
Создать чистый, масштабируемый и удобный в использовании сервис для проведения хакатонов с акцентом на удобство как для организаторов, так и для участников.
